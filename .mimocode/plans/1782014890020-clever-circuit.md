# Fix: Login Error Messages Not Showing

## Root Cause
The `<Toaster />` component (from sonner) is only mounted in `app/(dashboard)/layout.tsx:23`. The auth layout at `app/(auth)/layout.tsx` has no `<Toaster />`, so all `toast.error()` / `toast.success()` calls on login/register/forgot-password/verify-otp pages are silently swallowed.

## Fix
Add `<Toaster />` to `app/(auth)/layout.tsx` — same pattern as the dashboard layout.

### File: `app/(auth)/layout.tsx`
1. Add `"use client"` directive (Toaster requires client context)
2. Import `Toaster` from `@/components/ui/sonner`
3. Render `<Toaster />` alongside `{children}`

## Verification
1. Start dev server: `npm run dev`
2. Go to `/login`
3. Enter wrong credentials → toast error should appear
4. Enter empty fields → inline validation errors should still work
5. Login success → toast success should appear before redirect
