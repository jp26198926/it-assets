# Plan: Dynamic Sidebar from Pages DB Records

## Overview
Replace the hardcoded sidebar menu items in `app-sidebar.tsx` with dynamic data fetched from the `pages` MongoDB collection. Only active (non-deleted) pages will be shown. Hierarchical parent/child relationships via `parent_id` will build collapsible sub-menus.

## Changes

### 1. Create icon lookup table — `lib/icon-map.ts` (NEW)
- Export a `Record<string, React.ComponentType<{ className?: string }>>` mapping string icon names to Lucide React components
- Include all icons currently used in the sidebar plus common ones users might create pages with
- Export a helper `getIcon(name: string)` that returns the component or a fallback icon (e.g. `FileStack`)

### 2. Add server action for sidebar pages — `lib/actions/page-actions.ts`
- Add `getSidebarPages()` function that:
  - Fetches pages with status "Active" (joins with PageStatus to filter)
  - Populates `parent_id` with the parent's name
  - Returns `Page[]` (already has id, name, path, icon, parent_id)

### 3. Modify sidebar component — `components/layout/app-sidebar.tsx`
- Remove the hardcoded `menuItems` array
- Remove unused icon imports (keep only `Server`, `ChevronRight`, `LayoutDashboard` as fallback)
- Add `useState` + `useEffect` to call `getSidebarPages()` on mount
- Build a tree structure from the flat pages array:
  - Pages with `parent_id === null` → top-level items
  - Pages with a `parent_id` → children of that parent
- Map `page.icon` string → React component via the icon lookup table
- Render the tree using the same existing sidebar markup (collapsible for parents, links for children)
- Handle loading state (show skeleton or empty menu while fetching)

### 4. Files Modified
| File | Action |
|------|--------|

| `lib/actions/page-actions.ts` | Edit — add `getSidebarPages()` |
