# Plan: Create UOM (Unit of Measure) Module

## MongoDB Schema (UOM)

```
uoms collection:
  _id: ObjectId (auto)
  code: string (unique, required)
  name: string (unique, required)
  status: "Active" | "Deleted" (default: "Active")
  created_at: Date
  created_by: ObjectId | null
  updated_at: Date | null
  updated_by: ObjectId | null
  deleted_at: Date | null
  deleted_by: ObjectId | null
  deleted_reason: string | null
```

## Files to Create

- `lib/types/uom.ts` — UOM, CreateUOMInput, UpdateUOMInput, UOMFilters, UOMAdvancedFilter

## Reference Pattern
All files follow the existing **Department** module pattern exactly:
- Schema uses `BaseAuditSchemaDefinition` from `lib/db/base-schema.ts`
- Service uses `toUOM()` mapper and `connectDB()`
- API uses `withPageAuth("/uoms", permission)` for auth
- UI uses `PageGuard`, `ScrollReveal`, `sonner` toasts
- Columns use `useAuthorization()` for permission checks
- Page path: `/uoms`
