# Plan: Create "Category" Module

## Overview
Create a full CRUD module for **Categories** following the exact entity pattern established by UOMs/Permissions/Departments. The entity has a `name` (unique), `type` (enum: "Inventoriable" | "consumable"), optional `description`, and standard status/audit fields.

## Files to Create (16 files)

### Layer 1: TypeScript Types
- `lib/types/category.ts` — interfaces: `Category`, `CreateCategoryInput`, `UpdateCategoryInput`, `CategoryFilters`, `CategoryAdvancedFilter`

### Layer 2: Mongoose Model
- `lib/db/models/category.ts` — `ICategory` interface extending `BaseAuditFields`, schema with `name` (unique), `type` (enum), `description`, `status`, indexes on `name`, `type`, `status`

### Layer 3: Service
- `lib/services/category-service.ts` — `getCategories`, `getCategoryById`, `createCategory`, `updateCategory`, `deleteCategory`, `restoreCategory` + `toCategory` mapper

### Layer 4: Server Actions
- `lib/actions/category-actions.ts` — thin `"use server"` wrappers

### Layer 5: API Routes (4 files)
- `app/api/categories/route.ts` — GET (list with filters) + POST (create)
- `app/api/categories/[id]/route.ts` — GET (single) + PUT (update) + DELETE (soft-delete)
- `app/api/categories/[id]/restore/route.ts` — POST (restore)
- `app/api/categories/statuses/route.ts` — GET (list statuses)

### Layer 6: UI Components (7 files)
- `app/(dashboard)/categories/page.tsx` — dashboard page with PageGuard, state management, event handlers
- `components/data-table/category-data-table.tsx` — TanStack table with desktop table + mobile card views
- `components/data-table/category-data-table-columns.tsx` — column definitions (name, type, description, status, created_at, actions)
- `components/data-table/category-data-table-toolbar.tsx` — search, add button, advanced search/filter, export, column toggle
- `components/data-table/category-advanced-search-dialog.tsx` — server-side search by name, type
- `components/data-table/category-advanced-filter-dialog.tsx` — client-side advanced filters
- `components/data-table/category-export-buttons.tsx` — PDF + Excel export
- `components/modals/category-form-modal.tsx` — create/edit dialog with name (required), type (select, required, default "Inventoriable"), description (optional textarea)
- `components/modals/category-view-modal.tsx` — read-only detail dialog

## Key Design Decisions

### Type Enum Handling
- Mongoose schema: `type: { type: String, required: true, enum: ["Inventoriable", "Consumable"], default: "Inventoriable" }`
- TypeScript: `type: "Inventoriable" | "Consumable"` (both capitalized per user preference)
- Form modal: `<select>` dropdown with both options
- Columns: colored badge (Inventoriable = blue, Consumable = orange/amber)

### Search & Filter
- `search` in filters: `$regex` across `name` and `description`
- `type` filter: exact match enum
- `status` filter: exact match

### Auth Page Path
- `/categories` (consistent with URL convention)

### Sidebar Navigation
- **User will handle manually** — insert a page record in the `pages` collection:
  - `name`: "Categories"
  - `path`: "/categories"
  - `icon`: "Tag" (available in `lib/icon-map.ts`)
  - `parent_id`: null (top-level) or appropriate parent
