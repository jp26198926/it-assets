# Plan: Add Cloudinary upload button to Image URL field in item-form-modal.tsx

## Goal
Replace the plain text "Image URL" input with an input + Upload button that uploads images to Cloudinary. After upload, the resulting URL populates the input.

## Existing Patterns
- Cloudinary credentials are stored in MongoDB, loaded at runtime via `getCloudinarySettings()`
- Upload pattern: client converts file to base64 → server action uploads to Cloudinary → returns `secure_url`
- Existing example: `uploadProfilePhoto()` in `lib/actions/auth-actions.ts:218-243`
- Icons: project uses `lucide-react` (check existing imports in modal files)

## Implementation

### 1. Create server action `uploadItemImage` in `lib/actions/item-actions.ts`
Add a new exported function following the same pattern as `uploadProfilePhoto`:
- Accepts `fileBase64: string`, `fileName: string`
- Loads Cloudinary settings from DB, configures SDK
- Uploads to folder `it-assets/items` with public_id `item_${Date.now()}_${sanitizedFileName}`
- Returns `{ success, url?, error? }`

### 2. Update `components/modals/item-form-modal.tsx`
- Import `uploadItemImage` from `@/lib/actions/item-actions`
- Import `Upload` icon from `lucide-react` and `useToast` from `@/hooks/use-toast`
- Add state: `uploadingImage` (boolean)
- Add hidden `<input type="file" accept="image/*">` ref
- Replace the Image URL `<Input>` with a flex row:
  - `<Input>` for the URL (editable, placeholder stays)
  - `<Button type="button" variant="outline" onClick={...}>` with Upload icon
- On button click: trigger hidden file input
- On file select: validate type (image/*) and size (5MB max), convert to base64 via FileReader, call `uploadItemImage`, set `formData.image_url` to returned URL
- Show upload spinner on button while uploading

### Files to modify
- `lib/actions/item-actions.ts` — add `uploadItemImage` function
- `components/modals/item-form-modal.tsx` — add upload UI and logic
