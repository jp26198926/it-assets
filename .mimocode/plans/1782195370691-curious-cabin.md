# Plan: Update Add New Asset Modal

## Requirements
1. "Item" dropdown: full width, positioned above Barcode and Serial Number
2. "Barcode" field: editable (already is — auto-generated default but user can type over it)
3. Remove "Status" field from the form
4. Add "Remarks" field (textarea)

## Files to Modify

### 1. `lib/db/models/asset.ts` — Add `remarks` to Mongoose schema
- Add `remarks: { type: String, default: null }` to the schema
- Add `remarks: string | null` to `IAsset` interface

### 2. `lib/types/asset.ts` — Add `remarks` to TypeScript types
- Add `remarks: string | null` to `Asset` interface
- Add `remarks?: string` to `CreateAssetInput` interface

### 3. `lib/services/asset-service.ts` — Wire `remarks` through service layer
- In `toAsset()`: add `remarks: (d.remarks as string) ?? null`
- In `createAsset()`: add `remarks: data.remarks || null`
- In `updateAsset()`: add `if (data.remarks !== undefined) updateData.remarks = data.remarks || null`

### 4. `components/modals/asset-form-modal.tsx` — Main form changes
- Remove `statuses` constant and `status` from `defaultFormData`
- Remove `status` from the `useEffect` that populates edit data
- **Reorder fields**: Move the Item `<div className="space-y-2">` (currently inside a `grid grid-cols-2`) out to its own full-width row at the top, above Barcode/Serial Number
- Remove the entire Status `<Select>` block
- Add a full-width "Remarks" `<Textarea>` field (import `Textarea` from `@/components/ui/textarea`)
- In `handleSubmit`: add `remarks` to the submitted data, remove `status` handling
- In `defaultFormData`: remove `status: "Available"`, add `remarks: ""`

### 5. `components/modals/asset-view-modal.tsx` — Display Remarks
- Add a "Remarks" field in the left column of the main details grid

### 6. `app/api/assets/route.ts` — POST handler
- Add `remarks: body.remarks` to the `createAsset()` call

### 7. `app/api/assets/[id]/route.ts` — PUT handler
- Add `remarks: body.remarks` to the `updateAsset()` call

## Implementation Order
1. Model → 2. Types → 3. Service → 4. API routes → 5. Form modal → 6. View modal
