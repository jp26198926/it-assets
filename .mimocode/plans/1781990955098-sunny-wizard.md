# Show Avatar in Profile Modal and Header After Save

## Problem
`AuthUser` type lacks `avatar_url`. Neither `getCurrentUser` nor `updateMyProfile` return it. Header always shows initials.

## Files to Modify
1. `lib/types/auth.ts` — add `avatar_url` to `AuthUser`
2. `lib/services/auth-service.ts` — return `avatar_url` in `getCurrentUser`
3. `lib/actions/auth-actions.ts` — return `avatar_url` in `updateMyProfile`
4. `components/modals/profile-modal.tsx` — use `user.avatar_url` as initial avatar
5. `components/layout/header.tsx` — show avatar image when available

## Changes

### 1. `lib/types/auth.ts`
Add `avatar_url: string | null` to `AuthUser` interface.

### 2. `lib/services/auth-service.ts` (`getCurrentUser`)
- Add `avatar_url: user.avatar_url ?? null` to both the fallback and normal return objects.

### 3. `lib/actions/auth-actions.ts` (`updateMyProfile`)
- Add `avatar_url: updatedUser.avatar_url ?? null` to the returned user object.

### 4. `components/modals/profile-modal.tsx`
- In the `useEffect` that initializes state from `user`, set `setAvatarUrl(user.avatar_url)` instead of `setAvatarUrl(null)`.

### 5. `components/layout/header.tsx`
- In the avatar display (line 140), check `user.avatar_url`:
  - If present: render `<img>` with the avatar URL
  - If absent: render initials as before

## Verification
1. `npm run lint` — no new errors
2. Open profile modal → see current avatar (or initials if none)
3. Upload/take photo → save → avatar updates in modal
4. Close modal → header shows the new avatar
5. Refresh page → avatar persists in header
