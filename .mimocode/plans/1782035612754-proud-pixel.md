# Items Module Implementation Plan

## Overview
Create a full CRUD "Items" module following the existing Entity Pattern, with FK references to `categories` and `uoms` collections.

## MongoDB Schema (items collection)

```typescript
{
  name: { type: String, required: true, unique: true },
  category_id: { type: Schema.Types.ObjectId, ref: "Category", default: null },
  brand: { type: String, default: null },
  model: { type: String, default: null },
  description: { type: String, default: null },
  uom_id: { type: Schema.Types.ObjectId, ref: "UOM", default: null },
  minimum_stock: { type: Number, default: 0 },
  image_url: { type: String, default: null },
  status: { type: String, enum: ["Active", "Deleted"], default: "Active" },
  ...BaseAuditSchemaDefinition
}
```

## Files to Create (20 files)

### 1. Types ‚Äî `lib/types/item.ts`
- `Item` interface: `id`, `name`, `category_id: string | null`, `category_name?: string`, `brand: string | null`, `model: string | null`, `description: string | null`, `uom_id: string | null`, `uom_name?: string`, `minimum_stock: number`, `image_url: string | null`, `status`, audit fields
- `CreateItemInput`: `name` (required), rest optional
- `UpdateItemInput`: `Partial<CreateItemInput>`
- `ItemFilters`: `search?`, `name?`, `category_id?`, `brand?`, `model?`, `uom_id?`, `status?`
- `ItemAdvancedFilter`

### 2. Mongoose Model ‚Äî `lib/db/models/item.ts`
- `IItem extends Document, BaseAuditFields`
- Schema with all fields, indexes on `name` (unique), `status`, `category_id`, `uom_id`
- Model export with `mongoose.models.Item || mongoose.model<IItem>("Item", ItemSchema, "items")`

### 3. Service ‚Äî `lib/services/item-service.ts`
- `toItem()` converter with populated FK handling for both `category_id` and `uom_id` (same pattern as Employee‚ÜíDepartment)
- `getItems(filters?)` ‚Äî list with search, populate `category_id` and `uom_id`
- `getItemById(id)` ‚Äî single with populate
- `createItem(data)` ‚Äî create + re-fetch
- `updateItem(id, data)` ‚Äî update + re-fetch with populate
- `deleteItem(id, reason?)` ‚Äî soft delete
- `restoreItem(id)` ‚Äî restore
- `getItemSelectOptions()` ‚Äî returns `{ categories: {id, name}[], uoms: {id, name, code}[] }` for form dropdowns

### 4. Server Actions ‚Äî `lib/actions/item-actions.ts`
- 7 thin wrappers: `getItems`, `getItemById`, `createItem`, `updateItem`, `deleteItem`, `restoreItem`, `getItemSelectOptions`

### 5. API Routes (4 files)

- `app/api/items/[id]/route.ts` ‚Äî GET + PUT + DELETE

### 6. Dashboard Page ‚Äî `app/(dashboard)/items/page.tsx`

### 8. Modals (2 files)

## Key Design Decisions
- FK fields (`category_id`, `uom_id`) use `.populate()` in service layer, with converter handling both populated object and string ID forms (same pattern as Employee‚ÜíDepartment)
- Form modal fetches dropdown options via `getItemSelectOptions()` on open
- `image_url` is a text field (URL) ‚Äî no file upload in this phase
- `minimum_stock` defaults to 0, rendered as number in table
- Search filter covers: `name`, `brand`, `model`, `description`
