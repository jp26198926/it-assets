# Plan: Asset DataTable Column Changes

## Summary
Modify the Assets datatable to remove "Created" and "Location" columns, and add "Brand", "Model", and "Category" columns populated from the item/category collections.

## Data Chain
```
Category (name) ← Item (category_id → Category) ← Asset (item_id → Item)
```
Item has `brand`, `model`, and `category_id` fields. Category has `name`.

---

## Changes

### 1. Add new fields to `Asset` type
**File:** `lib/types/asset.ts`
- Add `item_brand?: string`
- Add `item_model?: string`
- Add `item_category_name?: string`

### 2. Update asset service populate + mapper
**File:** `lib/services/asset-service.ts`

**Populate** — change `.populate("item_id", "name")` to:
```
.populate({ path: "item_id", select: "name brand model category_id", populate: { path: "category_id", select: "name" } })
```
Apply this to all 4 query sites: `getAssets`, `getAssetById`, `createAsset` (after create), `updateAsset`.

**toAsset() mapper** — after the existing item extraction block (lines 11-23), add extraction of `brand`, `model`, and nested `category_id.name` from the populated item object:
```ts
let item_brand: string | undefined;
let item_model: string | undefined;
let item_category_name: string | undefined;
if (itemId && typeof itemId === "object" && "_id" in itemId) {
  item_brand = itemId.brand ?? undefined;
  item_model = itemId.model ?? undefined;
  const cat = itemId.category_id;
  if (cat && typeof cat === "object" && "name" in cat) {
    item_category_name = cat.name;
  }
}
```
Add these 3 fields to the return object.

## Bug Fix: MissingSchemaError for Category (round 2)

The import was already added but the bundler tree-shakes it because `CategoryModel` is never referenced in code. Mongoose populates `ref: "Category"` by string lookup, so the model must be registered as a side-effect.

**Fix:** Add a side-effect reference `CategoryModel;` in the `getAssets` function body to prevent tree-shaking.

In `getAssets()` at the top of the function (after `await connectDB();`), add:
```ts
void CategoryModel;
```

This ensures the import is preserved and the Category model is registered with Mongoose before the nested populate runs.
