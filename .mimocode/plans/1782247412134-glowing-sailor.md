# Plan: Create "Ticket Categories" Module

## Overview
Create a new "Ticket Categories" CRUD module following the exact same pattern as the existing **Category** module. The entity is simpler — only `name` and `status` fields (no `type` or `description`).

## Schema (MongoDB)
```
name: String, required, unique
status: String, enum ["Active", "Deleted"], default "Active"
+ BaseAuditFields (created_at, created_by, updated_at, updated_by, deleted_at, deleted_by, deleted_reason)
```

## Files to Create (20 files)

### 1. Types — `lib/types/ticket-category.ts`
- `TicketCategory` interface (id, name, status, audit fields + `*_name` display fields)
- `CreateTicketCategoryInput` (name)
- `UpdateTicketCategoryInput` (Partial of CreateInput)
- `TicketCategoryFilters` (search, name, status)

### 2. Mongoose Model — `lib/db/models/ticket-category.ts`
- Extends `BaseAuditFields`
- Fields: `name` (String, required, unique), `status` (String, enum, default "Active")

## Key Differences from Category
1. **No `type` field** — TicketCategory only has `name` and `status`
2. **No `description` field** — simplified entity
3. **Collection name** — `ticket_categories` (not `categories`)
4. **Page path** — `/ticket-categories`
5. **Route path** — `/api/ticket-categories`
6. **Page title** — "Ticket Categories"
7. **Permission path** — `/ticket-categories`
