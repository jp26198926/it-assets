## Changes

### 1. Fetch asset details for Section 3

The ticket only has `asset_id` and `asset_name`. To get serial_number, item_brand, item_model, we need to call `getAssetById` from `@/lib/actions/asset-actions`.

- Add import: `import { getAssetById } from "@/lib/actions/asset-actions";`
- Add state: `const [assetDetails, setAssetDetails] = useState<{ serial_number: string | null; item_brand: string | null; item_model: string | null } | null>(null);`
- In the `useEffect`, after fetching ticket, if `data?.asset_id` exists, call `getAssetById(data.asset_id)` and store the result.
- In Section 3 of the PDF, expand from just "Asset Name" to show:
  - Asset Name (`ticket.asset_name`)
  - Serial Number (`assetDetails?.serial_number`)
  - Brand (`assetDetails?.item_brand`)
  - Model (`assetDetails?.item_model`)

### 2. Add image rendering helper

Create an async helper function `fetchImageAsBase64(url: string): Promise<string | null>` that:
- Fetches the image via `fetch(url)`
- Converts response to blob → base64 data URI via FileReader
- Returns the data URI string (or null on failure)

Create a helper `extractImagesFromHtml(html: string): string[]` that:
- Parses HTML with DOMParser
- Extracts all `<img>` `src` attributes
- Returns array of image URLs

Create a helper `extractTextFromHtml(html: string): string` that:
- Parses HTML with DOMParser
- Extracts text content (preserving line breaks between elements)
- Returns clean text string

### 3. Update Section 4 — Description with images

Replace the current plain-text description rendering with:
1. Extract text from `ticket.description` HTML
2. Extract image URLs from the HTML
3. Render text using `doc.splitTextToSize` + `doc.text`
4. For each image URL: fetch as base64, calculate dimensions (maintain aspect ratio, max width = `cw`), call `doc.addImage()` at the current y position, advance y

This requires making `handlePrintPdf` async (it already calls async `getAppSettings` but the function isn't marked async — need to verify). The PDF generation itself can remain synchronous since we'll pre-fetch images.

**Approach**: Pre-fetch all images before starting PDF generation. Collect them in a map: `Map<string, { dataUri: string; width: number; height: number }>`. Then render inline during PDF construction.

Add an async phase at the start of `handlePrintPdf`:
```ts

const allImageUrls = [
  ...extractImagesFromHtml(ticket.description),
  ...comments.flatMap(c => extractImagesFromHtml(c.message)),
];
// Fetch all in parallel
const imageResults = await Promise.allSettled(
  allImageUrls.map(url => fetchImageAsBase64(url))
```
