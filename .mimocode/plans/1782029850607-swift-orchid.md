# Fix: "Add Page" button not showing despite having Add permission

## Root Cause

The authorization service (`lib/services/authorization-service.ts:12`) has a 5-minute in-memory cache (`CACHE_TTL = 5 * 60 * 1000`) for role permissions. The `invalidateCache()` function exists (line 96) but is **never called** when role permissions are added or removed.

### Flow:
1. User logs in → `AuthorizationProvider` fetches permissions → cached server-side for 5 min
2. Admin adds "Add" permission to the role via `role-permission-service.ts`
3. `addRolePermission()` does `$push` to DB but **does not invalidate cache**
4. When the user refreshes, `getUserPagePermissions()` returns **stale cached permissions** (without "Add") for up to 5 minutes

### Files involved:
- `lib/services/authorization-service.ts:12,60-75,96-101` — cache + `invalidateCache()`
- `lib/services/role-permission-service.ts:8-9,12-13` — `createRolePermission` / `deleteRolePermission` (no cache invalidation)
- `lib/services/role-service.ts:117-136,138-147` — `addRolePermission` / `removeRolePermission` (no cache invalidation)
- `components/data-table/page-data-table-toolbar.tsx:40` — `hasPermission("/pages", "Add")` check

## Fix

Call `authorizationService.invalidateCache(roleId)` in `role-permission-service.ts` after both `createRolePermission` and `deleteRolePermission`.

### File to modify:
- **`lib/services/role-permission-service.ts`**

Add import and cache invalidation calls:
```typescript
import * as authorizationService from "@/lib/services/authorization-service";

export async function createRolePermission(...): Promise<void> {
  await roleService.addRolePermission(roleId, data.page_id, data.permission_id);
  authorizationService.invalidateCache(roleId);
}

export async function deleteRolePermission(...): Promise<void> {
  await roleService.removeRolePermission(roleId, pageId, permissionId);
  authorizationService.invalidateCache(roleId);
}
```

## Verification

1. Run `npm run dev` (or restart dev server)
2. Log in as a user with a role that does NOT have "Add" permission for Pages
3. Verify the "Add Page" button is hidden
4. In another session/tab, go to Roles → Permissions → add "Add" permission to the Pages page for that role
5. Refresh the first tab — the "Add Page" button should now appear immediately (no 5-min wait)
