# Plan: Fix Print Layout Not Displaying

## Problem
The print-only corporate form is not rendering when printing. The `@media print` override for `.print-only` (set to `display: block !important`) is being defeated by Tailwind CSS v4's layer processing, which may reorder or deprioritize the override.

## Root Cause
Tailwind CSS v4's `@import "tailwindcss"` uses CSS layers. Custom CSS outside `@layer` blocks has different specificity ordering than layered CSS. The `@media print` rule for `.print-only` may be processed with lower priority than expected.

## Solution
Add an inline `<style jsx global>` tag inside the component with the `@media print` override. This bypasses Tailwind's layer processing entirely and guarantees the print styles take effect.

## Steps

### 1. Add inline print styles to the page component

### 2. Remove the redundant print CSS from globals.css
**File:** `app/globals.css`

Remove the `@media print` block entirely (lines 156-378) and the `.print-only` rule (lines 152-154). All print styles will now live inside the component's inline `<style>` tag, avoiding Tailwind layer conflicts.

Keep only the non-print utility classes (`.hover-lift`, etc.) in globals.css.
