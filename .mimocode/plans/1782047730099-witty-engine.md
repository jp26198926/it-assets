# Update Assignment Form Modal

## Goal
Revamp the "Add New Assignment" form modal:
1. **Asset** — searchable dropdown (select by barcode/name) instead of raw ID input
2. **Assign To** — radio toggle: either Employee OR Department (not both simultaneously)
3. **Remove** Status, Returned Date, and Condition on Return fields from the create/edit form

## Files to Modify

### 1. `lib/services/assignment-service.ts`
- Add `getAssignmentSelectOptions()` function that returns:
  - `assets`: `{ id, barcode, itemName }[]` — only assets with status "Available" (not assigned/repair/lost/disposed)
  - `employees`: `{ id, name }[]` — active employees
  - `departments`: `{ id, name }[]` — active departments

### 2. `lib/actions/assignment-actions.ts`
- Export `getAssignmentSelectOptions()` server action wrapping the service function

### 3. `components/modals/assignment-form-modal.tsx` (main change)
- Load assets/employees/departments via `getAssignmentSelectOptions()` on modal open
- **Asset field**: Combobox-style — type to search barcode, select from filtered dropdown
- **Assign To section**: Two radio-style buttons ("Employee" / "Department") with mutually exclusive select dropdowns
- **Remove** Status, Returned Date, Condition on Return fields from the form
- Keep: Asset (required), Assigned Date (required), Condition on Issue (required), Remarks (optional)
- Default `status` to `"Active"` in submit (not shown in form)

### 4. `lib/types/assignment.ts`
- No schema changes needed — fields stay, just hidden from form

## Implementation Details

### Asset Dropdown
- State: `assetSearch`, `assets[]`, `selectedAssetDisplay`
- On open: fetch all available assets
- Input shows barcode (e.g., `IT250001`), with dropdown showing barcode + item name
- On select: set `asset_id` in formData

### Assign To Toggle
- State: `assignTo: "employee" | "department"`, `employees[]`, `departments[]`
- Two styled buttons side by side
- Selecting "Employee" shows employee dropdown, clears `department_id`
- Selecting "Department" shows department dropdown, clears `employee_id`
- Default: no selection (both `employee_id` and `department_id` null)

### Form Layout
```
Asset *          | Assigned Date *
[Search asset...] | [date picker]

Assign To:
[Employee] [Department]

[Employee/Department dropdown based on selection]

Condition on Issue *
[text input]

Remarks
[textarea]
```

## New Task: Auto-fill Department from Employee

When "Employee" is selected in Assign To and an employee is chosen, auto-fill `department_id` based on that employee's department.

### Changes

**1. `lib/services/assignment-service.ts`**
- Update `getAssignmentSelectOptions()`: populate `department_id` on employees, return `departmentId` in the employees array
- Change employees return type from `{ id, name }` to `{ id, name, departmentId }`

**2. `lib/actions/assignment-actions.ts`**
- Update the return type of `getAssignmentSelectOptions` to match

**3. `components/modals/assignment-form-modal.tsx`**
- Update employees state type to include `departmentId`
- In the employee `Select` `onValueChange`: look up the selected employee's `departmentId` and set both `employee_id` and `department_id` in formData
