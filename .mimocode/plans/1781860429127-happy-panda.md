# Departments CRUD Page - Implementation Plan

## Overview
Create a full CRUD page for "Departments" following the established entity pattern. The Mongoose model already exists (`lib/db/models/department.ts` and `lib/db/models/department-status.ts`), so we only need the service, actions, types, API routes, and UI components.

**Department fields**: `code` (unique), `name` (unique), `description` (nullable), `status_id`, plus audit fields.

## Key Differences from Permission Entity
- **Extra field**: `code` (required, unique) - appears before `name` in forms/tables
- **Filter fields**: search includes `code`, `name`, `description`; advanced search/filter adds `code`
- **Columns**: `code` column added (first column), followed by `name`, `description`, `status`, `created_at`, actions
- **Form modal**: 3 fields (code, name, description) vs permission's 2
- **View modal**: shows `code` field
- **Advanced search**: includes code field
- **Advanced filter**: includes code field
- **Export**: includes Code column in PDF/Excel headers
