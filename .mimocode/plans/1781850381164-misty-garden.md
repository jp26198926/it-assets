# Plan: Create Permissions Page

## Context
The user wants a "Permissions" page with the same functionality and layout as the existing "Pages" page. The Permission model already exists in the database (`lib/db/models/permission.ts`) with fields: `name` (unique), `description`, `status_id`. The `PermissionStatus` model and seed are also already in place. We need to create the full CRUD UI layer.

## Permission Model (already exists)
- `name`: string (unique, required)
- `description`: string | null
- `status_id`: ObjectId ‚Üí PermissionStatus
- Base audit fields: `created_at`, `created_by`, `updated_at`, `updated_by`, `deleted_at`, `deleted_by`, `deleted_reason`
