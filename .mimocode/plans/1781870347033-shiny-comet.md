# Cloudinary Settings Page — Implementation Plan

## Overview
Create a Cloudinary settings page following the same singleton pattern as SMS/Mail. 3-field settings entity with GET + PUT API routes, disclaimer, and upload test feature.

## Files to Create

### 1. Type definitions — `lib/types/cloudinary.ts` (NEW)
- `Cloudinary` interface (fields + audit fields)
- `UpdateCloudinaryInput` interface

Fields:
- `cloud_name: string` (required in form)
- `api_key: string` (required in form, password input)
- `api_secret: string` (required in form, password input)

### 2. Mongoose model — `lib/db/models/cloudinary.ts` (NEW)
- `ICloudinary extends Document, BaseAuditFields`
- Collection name: `"cloudinary"`
- 3 string fields with default ""

### 3. Service — `lib/services/cloudinary-service.ts` (NEW)
- `toCloudinary()` converter
- `getCloudinarySettings()` — finds or creates singleton
- `updateCloudinarySettings(data)` — partial update
- `testCloudinaryUpload(fileBuffer, fileName)` — uploads a test file to Cloudinary using the `cloudinary` npm package (sign request with api_key + api_secret + cloud_name)

### 4. Install cloudinary
- `npm install cloudinary @types/cloudinary`

### 5. Server actions — `lib/actions/cloudinary-actions.ts` (NEW)
- `getCloudinarySettings()`, `updateCloudinarySettings(data)`, `testCloudinaryUpload(fileBase64, fileName)`

### 6. API routes
- `app/api/cloudinary/route.ts` (NEW) — GET + PUT
- `app/api/cloudinary/test/route.ts` (NEW) — POST for test upload

### 7. UI Page — `app/(dashboard)/cloudinary/page.tsx` (NEW)
- Same sticky header, card layout, floating save button
- Cards:
  1. **Cloudinary Configuration** (full-width) — cloud_name, api_key (password), api_secret (password)
  2. **Disclaimer** (full-width, blue card with Info icon)
  3. **Test Upload** (full-width) — file input + "Upload Test" button + preview of uploaded image

## Verification
1. Run `npm run dev` and navigate to `/cloudinary`
2. Fill in fields → Save → verify toast
3. Reload → verify persisted
4. Upload a test image → verify success + preview
5. Check API: `GET /api/cloudinary`, `PUT /api/cloudinary`, `POST /api/cloudinary/test`
6. Run `npm run lint`
