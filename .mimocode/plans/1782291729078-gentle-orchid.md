# Asset Detail Page `/assets/[id]`

## Overview
Create a dedicated detail page for individual assets at `/assets/[id]`, replacing the current modal-based view. The "View Details" action in the data table will navigate to this page. The page shows the full asset details, assignment history, and ticket history ŌĆö inspired by the `/tickets/[id]` layout pattern.

### 4. Create the asset detail page

**File:** `app/(dashboard)/assets/[id]/page.tsx` (NEW)

A `"use client"` page component following the tickets/[id] layout pattern:

**Layout:** Two-column grid on desktop (`grid-cols-1 lg:grid-cols-[1fr_340px]`)

**Left column (main content):**
- **Asset Header Card** ŌĆö barcode, status badge, item name badge, assigned-to info
- **Asset Details Card** ŌĆö 2-column grid showing all fields:
  - Left: Item, Barcode, Serial Number, Location, Assigned To, Department, Remarks, Date Received
  - Right: Purchase Date, Purchase Price (USD formatted), Warranty Expiry
- **Audit Fields Card** ŌĆö Created At, Created By, Last Updated, Updated By, conditional deleted fields

**Right column (sidebar, sticky):**
- **Action Buttons** ŌĆö Edit (navigates to open edit modal or separate edit page), Back to Assets
- **Asset Metadata** ŌĆö Status badge, Category, Location, Assigned To/Department
- **Assignment History** (Collapsible) ŌĆö Table/list of all assignments for this asset, fetched via `getAssignments({ asset_id })`. Shows: Employee, Department, Assigned Date, Returned Date, Status, Condition
- **Ticket History** (Collapsible) ŌĆö List of all tickets linked to this asset, fetched via `getTickets({ asset_id })`. Shows: Ticket No, Title, Status, Priority, Created Date

**Data fetching:**
- Use `getAssetById(id)` for asset details
- Use `getAssignments({ asset_id: id })` for assignment history
- Use `getTickets({ asset_id: id })` for ticket history
- Use `useParams()` for the asset ID
- Use `useRouter()` for navigation
- Use `useBreadcrumbOverrides()` to set the barcode as breadcrumb label
- Use `PageGuard` with `pagePath="/assets"` for auth

**Status configs:** Reuse same status color mapping as asset-view-modal.tsx
