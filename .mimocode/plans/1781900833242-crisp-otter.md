# Redesign Profile Modal: Show Current Phone + Send Code Button Flow

## Overview
Fix the Profile modal to show the current phone number from the database, and redesign the email/phone change flow: add "Send Code" buttons next to each field, OTP input appears inline when clicked, and "Save Changes" validates OTPs before saving.

---

## Files to Modify

1. `lib/types/auth.ts` — Add `phone` to `AuthUser`
2. `lib/services/auth-service.ts` — Include `phone` in `getCurrentUser` return
3. `components/modals/profile-modal.tsx` — Full redesign of email/phone change flow

---

## Implementation

### Step 1: Add `phone` to AuthUser Type
**Modify `lib/types/auth.ts`**
- Add `phone: string | null` to `AuthUser` interface

### Step 2: Include Phone in getCurrentUser
**Modify `lib/services/auth-service.ts`**
- In `getCurrentUser()`, add `phone: user.phone ?? null` to the returned object

### Step 3: Redesign Profile Modal
**Modify `components/modals/profile-modal.tsx`**

Key changes:
- **Phone field**: Show current phone from database (disabled, like email) instead of empty input
- **Change Phone field**: New input below current phone for entering new number
- **Send Code buttons**: Add "Send Code" button next to both "Change Email" and "Change Phone" inputs
- **OTP fields**: When "Send Code" is clicked, OTP input appears directly below the field
- **Save Changes**: Validates that OTPs have been verified before saving

**New state variables:**
- `currentPhone` — the phone from DB (loaded from `user.phone`)
- `emailVerified` — tracks if email OTP was verified (boolean)
- `phoneVerified` — tracks if phone OTP was verified (boolean)

**New flow:**
1. Modal opens → shows current email (disabled), current phone (disabled)
2. User types new email → clicks "Send Code" → OTP input appears below
3. User types new phone → clicks "Send Code" → OTP input appears below
4. User clicks "Save Changes":
   - If email changed but not verified → show error "Please verify your new email first"
   - If phone changed but not verified → show error "Please verify your new phone first"
   - If all verified or no changes → save profile + email + phone

**Layout (top to bottom):**
1. Photo upload
2. First Name / Last Name
3. Email Address (disabled, shows current)
4. Change Email input + "Send Code" button → OTP input below (when visible)
5. Phone Number (disabled, shows current)
6. Change Phone input + "Send Code" button → OTP input below (when visible)
