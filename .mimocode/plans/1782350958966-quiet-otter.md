# Ticket Email Notifications Plan

## Current State
- Email infrastructure exists: `lib/services/mail-service.ts` with nodemailer/SMTP
- Ticket service already sends emails for: creation, assignment, status changes (to requestor only), updates (to requestor only)
- **Missing**: No email notifications when comments/replies are added
- **Missing**: No email to technician when requestor makes changes
- Requestors can already submit replies and mark as resolved via "Submit as Resolved" button

## Requirements
1. When technician/admin replies → requestor gets notification email
2. When requestor replies → assigned technician gets email notification
3. When technician/admin updates status → requestor gets notification email (already exists)
4. When requestor updates status → assigned technician gets email notification
5. Technician should be notified on ALL ticket changes (replies, status, edits)

## Changes Required

### 1. Add Comment Notification Email Function
**File**: `lib/services/ticket-comment-service.ts`

Add a new function `sendCommentNotificationEmail` that sends email when a comment is created:
- If commenter is technician/admin → notify requestor (ticket.email)
- If commenter is requestor (Viewer) → notify assigned technician

### 2. Modify `createTicketComment` to Send Notifications
**File**: `lib/services/ticket-comment-service.ts`

After creating the comment, determine the commenter's role and send appropriate notification:
- Get the ticket details to find requestor email and assigned technician
- Check commenter role via user lookup
- Send email to the appropriate recipient

### 3. Add Email Notification to Technician on All Ticket Changes
**File**: `lib/services/ticket-service.ts`

Modify `updateTicket` to also notify the assigned technician when:
- Status changes (already notifies requestor, add technician notification)
- Any ticket field is updated (name, category, priority, etc.)
- Requestor is the one making the change

## Implementation Details

### Comment Notification Email Template
```typescript
async function sendCommentNotificationEmail(
  ticket_no: string,
  title: string,
  recipientEmail: string,
  recipientName: string,
  commenterName: string,
  messagePreview: string
): Promise<void>
```

### Logic Flow in `createTicketComment`
```typescript
// After creating comment:
const ticket = await TicketModel.findById(data.ticket_id).lean();
const commenter = await UserModel.findById(createdByUserId).lean();
const commenterRole = await RoleModel.findById(commenter.role_id).lean();

if (commenterRole.name === "Administrator" || commenterRole.name === "Technician") {
  // Notify requestor
  sendCommentNotificationEmail(ticket_no, title, ticket.email, ticket.name, commenterName, messagePreview);
} else if (commenterRole.name === "Viewer") {
  // Notify assigned technician
  if (ticket.assigned_to) {
    const technician = await UserModel.findById(ticket.assigned_to).lean();
    sendCommentNotificationEmail(ticket_no, title, technician.email, technicianName, commenterName, messagePreview);
  }
}
```

### Logic Flow in `updateTicket` (for technician notification)
```typescript
// After existing requestor notification:
if (oldAssignedTo && updatedByUserId !== oldAssignedTo) {
  const technician = await UserModel.findById(oldAssignedTo).lean();
  if (technician) {
    if (statusChanged) {
      sendTicketStatusChangedEmail(ticket_no, title, technician.email, oldStatus, newStatus, updatedByName);
    } else if (changes.length > 0) {
      sendTicketUpdatedEmail(ticket_no, title, technician.email, updatedByName, changes);
    }
  }
}
```
