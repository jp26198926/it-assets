# Plan: Add "Duplicate" Action to Roles DataTable

## Overview
Add a "Duplicate" action button to the Roles data table that opens a modal where the user can enter a new name and review the role's information (description + permissions) before creating a copy.

## Files to Modify

### 1. `lib/services/role-service.ts` — Add `duplicateRole` function
- Takes source role ID + new name/description
- Creates a new role via `RoleModel.create({ name, description, status: "Active" })`
- Fetches source role's permissions, then `$push`es them all into the new role
- Returns the new role with populated audit fields

### 2. `lib/actions/role-actions.ts` — Add `duplicateRole` server action
- Thin wrapper: `export async function duplicateRole(sourceRoleId: string, data: { name: string; description?: string }): Promise<Role>`

### 3. `components/modals/role-duplicate-modal.tsx` — New modal component
- Props: `open`, `onOpenChange`, `role: Role | null`, `onSubmit`
- On open: fetch the full role via `getRoleById(role.id)` to get populated permissions
- Fields: Name (required, pre-filled with `"Copy of {name}"`), Description (optional, pre-filled)
- Read-only section below fields showing the source role's permissions in a simple table (Page | Permission)
- Submit calls `onSubmit({ name, description })`

### 4. `components/data-table/role-data-table-columns.tsx` — Add "Duplicate" menu item
- Add `onDuplicate` callback to `ActionsProps` interface
- Add `Duplicate` icon import from lucide-react (use `Copy` icon)
- Add menu item between "Permission" and the separator, guarded by `canEdit` permission (same as edit)
- Thread `onDuplicate` through `createRoleColumns` params and `Actions` usage

### 5. `components/data-table/role-data-table.tsx` — Thread `onDuplicate` prop
- Add `onDuplicate: (role: Role) => void` to `RoleDataTableProps`
- Pass through to `RoleDataTableToolbar` and columns

### 6. `components/data-table/role-data-table-toolbar.tsx` — Pass `onDuplicate` if needed
- Check if toolbar needs it (likely not — only columns use it)

### 7. `app/(dashboard)/roles/page.tsx` — Wire up state + handler
- Add `duplicateRoleItem` state (`useState<Role | null>(null)`)
- Add `handleDuplicate(role)` handler: sets state, opens modal
- Add `handleDuplicateSubmit(data)`: calls `createRole(data)` then loops through permissions adding each via `createRolePermission`, or better — call a single `duplicateRole` server action
- Pass `handleDuplicate` to columns and data table
- Render `RoleDuplicateModal`

### 8. `app/api/roles/[id]/duplicate/route.ts` — New API route
- POST endpoint with `Add` permission check
- Accepts `{ name, description? }` in body
- Calls `roleService.duplicateRole(id, { name, description })`

## Duplicate Logic (in service layer)
```
1. Fetch source role by ID (throw if not found)
2. Create new role with name + description + status "Active"
3. Fetch source permissions via getRolePermissions(sourceRoleId)
4. For each permission: $push into new role's permissions array
5. Return new role with populated audit fields
```

## Fix: Sort permissions A-Z by Page in Duplicate Modal

### `components/modals/role-duplicate-modal.tsx`
Sort the `permissions` array by `page_name` before rendering in the table. Use `.sort((a, b) => (a.page_name || "").localeCompare(b.page_name || ""))`.
