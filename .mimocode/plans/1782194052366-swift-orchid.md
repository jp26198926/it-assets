# Fix: Scrollbar only on modal content (not header/footer)

## Problem
In `components/modals/asset-form-modal.tsx`, the `DialogFooter` is nested inside the scrollable `<form>` element (line 353). This causes the footer to scroll with the content instead of staying pinned at the bottom.

## Solution
Move `DialogFooter` outside the `<form>` to be a direct sibling inside `DialogContent`. Use the HTML5 `form` attribute on the submit button to associate it with the form by ID, since the button will no longer be a descendant of the `<form>`.

## File to modify
- `components/modals/asset-form-modal.tsx`

## Changes

### 1. Add `id="asset-form"` to the `<form>` element (line 153)
```tsx
<form id="asset-form" onSubmit={handleSubmit} className="space-y-4 flex-1 overflow-y-auto min-h-0">
```

### 2. Move `DialogFooter` outside the `</form>` closing tag
- Close the `</form>` before the `DialogFooter`
- Add `form="asset-form"` to the submit `<Button>` so it still triggers form submission

**Before (lines 350-366):**
```tsx
          {errors.submit && (
            <p className="text-sm text-red-500">{errors.submit}</p>
          )}
          <DialogFooter>
            <Button
              type="button"
              variant="outline"
              onClick={() => onOpenChange(false)}
              disabled={loading}
            >
              Cancel
            </Button>
            <Button type="submit" disabled={loading}>
              {loading ? "Saving..." : asset ? "Save Changes" : "Add Asset"}
            </Button>
          </DialogFooter>
        </form>
```

**After:**
```tsx
          {errors.submit && (
            <p className="text-sm text-red-500">{errors.submit}</p>
          )}
        </form>
        <DialogFooter>
          <Button
            type="button"
            variant="outline"
            onClick={() => onOpenChange(false)}
            disabled={loading}
          >
            Cancel
          </Button>
          <Button type="submit" form="asset-form" disabled={loading}>
            {loading ? "Saving..." : asset ? "Save Changes" : "Add Asset"}
          </Button>
        </DialogFooter>
```

## Result
```
DialogContent  flex flex-col max-h-[85vh]
  DialogHeader                          ← pinned top
  <form id="asset-form">  flex-1 overflow-y-auto min-h-0  ← scrolls
    ...fields + error messages...
  </form>
  DialogFooter                          ← pinned bottom (fixed)
    Cancel button
    Submit button (form="asset-form")
```
