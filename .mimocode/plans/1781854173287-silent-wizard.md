# Plan: Embed Role-Permissions in Role Schema

## Overview
Replace the separate `role_permissions` collection with an embedded `permissions` array inside the Role document. This simplifies the schema and makes role-permission management atomic.

## Current Schema
```
roles: { name, description, status_id }
role_permissions: { role_id, page_id, permission_id, status_id }
role_permission_statuses: { status }
```

## New Schema
```
roles: { name, description, status_id, permissions: [{ page_id, permission_id }] }
```
- No more `role_permissions` collection
- No more `role_permission_statuses` collection

## Files to Modify (7 files)

### 1. `lib/db/models/role.ts` — Add `permissions` subdocument array
Add `permissions` field to IRole interface and schema:
```ts
permissions: { page_id: Types.ObjectId; permission_id: Types.ObjectId }[]
```
Default to `[]`. No status_id needed per permission — they're either present or removed.

### 2. `lib/types/role.ts` — Add `RolePermissionEntry` type
Add:
```ts
export interface RolePermissionEntry {
  page_id: string;
  page_name?: string;
  permission_id: string;
  permission_name?: string;
}
```
Update `Role` interface to include `permissions: RolePermissionEntry[]`.

### 3. `lib/types/role-permission.ts` — Simplify
Simplify `CreateRolePermissionInput` to just `{ page_id, permission_id }` (no role_id — it's the parent).
Remove `RolePermissionFilters` (no longer a separate collection to query).

### 4. `lib/services/role-service.ts` — Add embedded permission operations
Add functions:
- `addRolePermission(roleId, pageId, permissionId)` — `$push` to permissions array
- `removeRolePermission(roleId, pageId, permissionId)` — `$pull` from permissions array
- Update `toRole()` to populate page/permission names from the embedded array

### 5. `lib/services/role-permission-service.ts` — Replace with role-service calls
Rewrite to call role-service functions instead of querying a separate collection:
- `getRolePermissions(roleId)` → fetch role, return populated permissions
- `createRolePermission(data)` → call `roleService.addRolePermission()`
- `deleteRolePermission(roleId, pageId, permissionId)` → call `roleService.removeRolePermission()`

### 6. `app/api/roles/[id]/permissions/route.ts` — Update to use role_id from URL
- `GET` — use `[id]` from params (not query param)
- `POST` — use `[id]` from params as role_id
- `DELETE` — use `[id]` from params + page_id/permission_id from body

### 7. `components/modals/role-permission-modal.tsx` — Update remove handler
- `handleRemove` now needs `page_id` + `permission_id` (not a record id)
- Update the table rows to pass these values
