# Plan: Responsive DataTable — Overflow-Based Card View Switching

## Problem
The DataTable clips content with `overflow-hidden` when the table is wider than the screen. The current Tailwind `lg:block`/`lg:hidden` approach switches based on viewport width, not actual content width. The user wants the table to **automatically switch to card view when its content overflows the screen**, without horizontal scrolling.

## Approach
Use a custom `useTableOverflow` hook that renders the table header off-screen with `width: max-content` to measure its natural width, compares it to the container width, and toggles between table and card view via state. A `ResizeObserver` handles dynamic resize.

## Files to Change

### 1. New: `hooks/use-table-overflow.ts` (~30 lines)
- Custom hook returning `{ containerRef, measureRef, isOverflowing, checkOverflow }`
- Off-screen measurement div with `width: max-content` captures true natural table width
- `ResizeObserver` on container watches for size changes
- `useIsomorphicLayoutEffect` for SSR-safe initial measurement
- Exposed `checkOverflow` callback for manual re-trigger

### 2. Modify: `components/data-table/data-table.tsx` (~30 lines changed)
- Remove `hidden lg:block` from table wrapper, `lg:hidden` from card wrapper
- Call `useTableOverflow()` hook
- Add `useEffect` on `columnVisibility` and `filteredData` to re-measure
- Render hidden measurement div with table header off-screen
- Toggle visibility: `isOverflowing ? "hidden" : "bg-white shadow-sm overflow-hidden"` for table
- Toggle visibility: `isOverflowing ? "space-y-3" : "hidden"` for cards

## Edge Cases Handled
- Window resize → `ResizeObserver`
- Sidebar toggle → same `ResizeObserver`
- Column visibility changes → `useEffect(columnVisibility)`
- Data/filter changes → `useEffect(filteredData)`
- Initial render → `useLayoutEffect` before paint
- SSR → `useIsomorphicLayoutEffect` fallback

## What NOT to Change
- `components/ui/table.tsx` — keep `overflow-hidden`
- `components/data-table/mobile-card.tsx` — works as-is
- `hooks/use-mobile.ts` — independent, not used
- `DataTableProps` interface — no new props needed
