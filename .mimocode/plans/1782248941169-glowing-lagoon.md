## Step 4: Service Layer — `lib/services/ticket-service.ts`

### Key functions:
1. **`getTickets(filters?)`** — Query with populate for category, asset, assigned_to, requestor, audit fields. Filter by all TicketFilters fields.
2. **`getTicketById(id)`** — Single ticket with full populates.
3. **`createTicket(data)`** — Auto-generate ticket_no, check if email exists in users collection:
   - If email found in users → set `requestor_id` to that user's _id
   - If email NOT found → auto-register user with:
     - `first_name`: extracted from email prefix (before @)
     - `last_name`: " " (space, since last_name is required)
     - `email`: the provided email
     - `password`: random 12-char string
     - `role_id`: Viewer role's _id (lookup by name "Viewer")
     - `status`: "Active"
     - Send welcome email with credentials via nodemailer (reuse mail settings from `getMailSettings()`)
   - Then set `requestor_id` to the new/existing user's _id
   - Create the ticket document
4. **`updateTicket(id, data, updated_by?)`** — Standard update with audit fields.
5. **`deleteTicket(id, reason?)`** — Soft delete.
6. **`restoreTicket(id)`** — Restore.
7. **`getTicketSelectOptions()`** — Return categories, assets, and users for dropdowns (sorted A-Z).

### `toTicket()` mapping function
Follow the same pattern as `toTicketCategory` — map populated audit fields (`created_by`, `updated_by`, `deleted_by`) from populated objects to `id + name`.

### Auto-registration email
Use nodemailer with settings from `getMailSettings()`. Send email with:
- Subject: "Welcome to IT Asset Manager"
- Body: Include the email and generated password

### `app/api/tickets/route.ts`

- **GET**: Single ticket
- **PUT**: Update ticket
