# Tickets Module — Role-Based Data Security

## Current State
The tickets page has page-level access control (`withPageAuth`) but **no data-level filtering**. All authenticated users with "Access" permission see ALL tickets.

## Requirements
| Role | Can See | Can Do |
|------|---------|--------|
| **Administrator** | All tickets | All actions (CRUD, delete, restore) |
| **Viewer** (Requester) | Only tickets where `requestor_id` = their userId | Create tickets, view own |
| **Technician** | Tickets where `created_by` = their userId OR `assigned_to` = their userId | Create tickets, update assigned tickets |

Enforce on both the **list view** and the **detail page** (`/tickets/[id]`).

## Implementation

### 1. `lib/services/ticket-service.ts` — Core filtering

**`getTickets(filters?, user?)`** — Add optional `user` param:
```ts
async function getTickets(filters?: TicketFilters, user?: { userId: string; role: string } | null): Promise<Ticket[]>
```
After building the existing query, append role-based filter:
- `Administrator` → no filter
- `Viewer` → `{ $or: [{ requestor_id: user.userId }, { created_by: user.userId }] }` (requesters see tickets they submitted OR created)
- `Technician` → `{ $or: [{ created_by: user.userId }, { assigned_to: user.userId }] }`

**`getTicketById(id, user?)`** — Add optional `user` param:
```ts
async function getTicketById(id: string, user?: { userId: string; role: string } | null): Promise<Ticket | null>
```
After fetching the ticket, validate access:
- `Administrator` → always allowed
- `Viewer` → `ticket.requestor_id === user.userId` OR `ticket.created_by === user.userId`
- `Technician` → `ticket.created_by === user.userId` OR `ticket.assigned_to === user.userId`
- Return `null` if not authorized (detail page already handles null → "Ticket not found")

### 2. `lib/actions/ticket-actions.ts` — Pass user context

- `getTickets()`: call `getAuthFromRequest()`, pass `{ userId, role }` to service
- `getTicketById()`: call `getAuthFromRequest()`, pass `{ userId, role }` to service

### 3. `app/api/tickets/route.ts` (GET) — Pass user from auth

Extract `auth.user` from `withPageAuth` result, pass to `getTickets()`.

### 4. `app/api/tickets/[id]/route.ts` (GET/PUT/DELETE) — Validate access

Extract `auth.user` from `withPageAuth` result, pass to `getTicketById()` for access check. Return 403 if null.

### 5. `app/(dashboard)/tickets/[id]/page.tsx` — No changes needed

Already handles `getTicketById` returning null → shows "Ticket not found" with back button. The access denial will naturally show this state.
