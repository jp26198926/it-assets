# Page Module Implementation Plan

## Overview
Create a "Page" module with CRUD operations that persists to MongoDB, following the same structure and styling as the Assets module.

## Current State
- **Page Model**: Defined at `lib/db/models/page.ts` with fields: name, description, path, icon, parent_id, section, status_id
- **PageStatus Model**: Defined at `lib/db/models/page-status.ts` with statuses: "Active", "Deleted"
- **Assets Module**: Uses client-side state with mock data, shadcn/ui components, TanStack Table

## Implementation Approach

### 1. Server Actions (`lib/actions/page-actions.ts`)
Create server actions for CRUD operations:

```typescript
"use server"

// Get all pages with optional filters
export async function getPages(filters?: PageFilters)

// Get single page by ID
export async function getPageById(id: string)

// Create new page (auto-assigns "Active" status)
export async function createPage(data: CreatePageInput)
// Implementation: Lookup "Active" status from PageStatus collection, assign to new page

// Update existing page
export async function updatePage(id: string, data: UpdatePageInput)

// Soft delete page (set deleted_at)
export async function deletePage(id: string, reason?: string)

// Restore deleted page
export async function restorePage(id: string)

// Get page status options (for filtering only, not for form)
export async function getPageStatuses()
```

### 2. TypeScript Types (`lib/types/page.ts`)
Define frontend types separate from Mongoose interface:

```typescript
export interface Page {
  id: string;
  name: string;
  description: string | null;
  path: string;
  icon: string;
  parent_id: string | null;
  section: string | null;
  status_id: string;
  status?: string; // populated from PageStatus
  created_at: Date;
  created_by: string | null;
  updated_at: Date | null;
  updated_by: string | null;
  deleted_at: Date | null;
}

export interface CreatePageInput {
  name: string;
  description?: string;
  path: string;
  icon: string;
  parent_id?: string;
  section?: string;
  status_id: string;
}

export interface UpdatePageInput extends Partial<CreatePageInput> {}

export interface PageFilters {
  search?: string;
  status?: string;
  section?: string;
  parent_id?: string;
}
```

2. **Soft Delete**: Use `deleted_at` field consistent with Page model design
3. **Parent Selection**: Flat dropdown list of all pages (simpler to implement and use)
4. **Icon Field**: Dropdown with Lucide icons and preview (curated list of ~30-40 common icons)
5. **Status Filter**: Dropdown with Active/Deleted options from PageStatus model

# Page Module: Search, Filter, and Export Features

## Overview
Add search, advanced search, filter, advanced filter, export to Excel, and export to PDF to the Pages module.

## Architecture

| Feature | Scope | Implementation |
|---------|-------|----------------|
| Search | Server-side (DB) | Wire toolbar search input to `getPages({ search })` server action |
| Advanced Search | Server-side (DB) | New `PageAdvancedSearchDialog` → calls `getPages()` with multi-field params |
| Filter | Client-side | `useMemo` filtering on loaded data (like assets) |
| Advanced Filter | Client-side | New `PageAdvancedFilterDialog` → `useMemo` AND logic on loaded data |
| Export to Excel | Client-side | `page-export-buttons.tsx` using `xlsx` library |
| Export to PDF | Client-side | `page-export-buttons.tsx` using `jspdf` + `jspdf-autotable` |

## Step 1: Server-Side Search Support

### 1a. Update `lib/actions/page-actions.ts`
Enhance `getPages` to support more search fields:
- `search`: regex on `name`, `path`, `description` (already exists)
- Add `name`, `path`, `description` as individual filter params for advanced search

### 1b. Update `lib/types/page.ts`
Add `PageAdvancedSearch` interface for multi-field server search:
```typescript
export interface PageAdvancedSearch {
  name?: string;
  path?: string;
  description?: string;
  section?: string;
}
```

## Step 2: Client-Side Filter Support

### 2a. Add `PageAdvancedFilter` type to `lib/types/page.ts`
```typescript
export interface PageAdvancedFilter {

  value: string;
}
```

- Add `advancedFilters` state
