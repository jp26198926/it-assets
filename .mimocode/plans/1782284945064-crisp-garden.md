# Plan: Add Department Field to Tickets Module

## Summary
Add an optional `department_id` field (referencing the `departments` collection) to the Tickets module across model, types, service, API, and UI layers.

---

## Files to Modify

### 1. Model — `lib/db/models/ticket.ts`
- Add `department_id: Types.ObjectId | null` to `ITicket` interface
- Add schema field: `department_id: { type: Schema.Types.ObjectId, ref: "Department", default: null }`
- Add index: `TicketSchema.index({ department_id: 1 })`

### 2. Types — `lib/types/ticket.ts`
- Add `department_id: string | null` and `department_name?: string` to `Ticket` interface
- Add `department_id?: string` to `CreateTicketInput`
- Add `department_id?: string` to `TicketFilters`

### 3. Service — `lib/services/ticket-service.ts`
- Import `Department as DepartmentModel` from `@/lib/db/models/department`
- **`toTicket()`**: Add department_id/department_name extraction (same pattern as `category_id`)
- **`populateQuery()`**: Add `.populate("department_id", "name")`
- **`createTicket()`**: Pass `department_id: data.department_id || null`
- **`updateTicket()`**: Handle `department_id` in updateData and changes tracking
- **`getTickets()`**: Add `department_id` filter support
- **`getTicketSelectOptions()`**: Add `departments` query (from DepartmentModel, active only, sorted A-Z) and return in result

### 4. Server Actions — `lib/actions/ticket-actions.ts`
- Update `getTicketSelectOptions()` return type to include `departments: { id: string; name: string }[]`

### 5. API Routes
- **`app/api/tickets/route.ts`**: Add `department_id` query param filter (GET), pass `department_id` in POST body
- **`app/api/tickets/[id]/route.ts`**: Pass `department_id` in PUT body

### 6. UI — Form Modal — `components/modals/ticket-form-modal.tsx`
- Add `departments` to `selectOptions` prop type
- Add `department_id` to `defaultFormData` and `CreateTicketInput` state
- Add Department `<Select>` dropdown (optional, "None" default) in the grid with Asset/AssignTo or in a new row

### 7. UI — View Modal — `components/modals/ticket-view-modal.tsx`
- Display `ticket.department_name` in the detail grid (right column, alongside Category/Asset/AssignedTo)

### 8. UI — Data Table Columns — `components/data-table/ticket-data-table-columns.tsx`
- Add `department_name` column (between Category and Priority or after Assigned To)

### 9. UI — Edit Modal — `components/modals/ticket-edit-modal.tsx`
- Add Department select to the edit modal's `selectOptions` and form state

### 10. UI — Export Buttons — `components/data-table/ticket-export-buttons.tsx`
- Add Department to PDF and Excel export rows

### 11. UI — Advanced Search — `components/data-table/ticket-advanced-search-dialog.tsx`
- Add department_id search/filter field

### 12. UI — Advanced Filter — `components/data-table/ticket-advanced-filter-dialog.tsx`
- Add `department_name` to `filterFields` array

---

## Key Patterns (from existing code)
- Department reference pattern: `Schema.Types.ObjectId, ref: "Department", default: null` (same as `lib/db/models/employee.ts:40-44`)
- Populate: `.populate("department_id", "name")` (same as employee-service.ts)
- toTicket extraction: same pattern as `category_id` — cast as `{ _id, name }` or string
- Select options: query `DepartmentModel.find({ deleted_at: null, status: "Active" })` sorted by name
