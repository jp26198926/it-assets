# Plan: Make table fluid/responsive тАФ fit viewport, word-wrap content

## Problem

The table currently has conflicting styles that prevent it from being truly fluid:

1. **`overflow-hidden` on table container** (`components/ui/table.tsx:11`) тАФ clips content instead of letting it flow. User explicitly said "not hide it if exceed."
2. **`max-w-[200px] truncate`, `max-w-[180px] truncate`, `max-w-[150px] truncate`** on column cells in `data-table-columns.tsx` тАФ `truncate` includes `white-space: nowrap` which prevents word-wrapping. These hardcoded max-widths fight against fluid layout.
3. **`table-layout: fixed`** on the table (`table.tsx:16`) тАФ forces equal-width columns. While this prevents overflow, it's not truly "fluid." The user wants the table to resize naturally with content.

## Approach

Use `table-layout: auto` (the natural default) with word-wrapping on all cells. This lets the browser auto-size columns based on content while `w-full` ensures the table fills its container. Remove all `truncate` and `max-w` constraints from column cells so content wraps naturally.

The table container wrapper in `data-table.tsx` keeps `overflow-hidden` only as a safety net тАФ with proper word-wrapping, it should never actually clip anything.

## Changes

### 1. `components/ui/table.tsx`

- **Container div (line 11)**: Change `overflow-hidden` тЖТ `overflow-hidden` (keep as safety net тАФ no change needed)
- **Table element (line 15-16)**: Remove `style={{ tableLayout: "fixed" }}` тАФ use default `auto` layout so columns auto-size based on content
- **TableCell (line 87)**: Add `overflow-wrap: break-word` alongside `break-words` for extra safety on long unbreakable strings

### 2. `components/data-table/data-table-columns.tsx`

Remove `truncate` and `max-w-*` from three columns so content word-wraps instead of being clipped:
