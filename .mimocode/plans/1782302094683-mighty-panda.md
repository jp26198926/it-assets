# Plan: Update Dashboard with Real Data, Charts, and Role-Based Access

## Goal
Replace the current mock-data dashboard with real database queries, add meaningful charts via Recharts, and conditionally show/hide dashboard sections based on user role permissions.

---

## 1. Create Dashboard Service (`lib/services/dashboard-service.ts`)

Server-side aggregation queries returning all data the dashboard needs in one call. Uses MongoDB aggregation pipeline on existing collections.

**Queries to implement:**

| Metric | Source | Aggregation |
|--------|--------|-------------|
| Total Assets | `assets` (non-deleted) | `countDocuments({ deleted_at: null })` |
| Assets by Status | `assets` (non-deleted) | `aggregate` group by `status` |
| Assets by Category | `assets` → `items` → `categories` | `aggregate` with `$lookup` chain |
| Recent Assets | `assets` (non-deleted) | `find().sort({ created_at: -1 }).limit(5)` with populated item name |
| Total Tickets | `tickets` (non-deleted) | `countDocuments({ deleted_at: null })` |
| Tickets by Status | `tickets` (non-deleted) | `aggregate` group by `status` |
| Tickets by Priority | `tickets` (non-deleted) | `aggregate` group by `priority` |
| Recent Tickets | `tickets` (non-deleted) | `find().sort({ created_at: -1 }).limit(5)` |
| Open Tickets | `tickets` | `countDocuments({ status: { $in: ["Open", "In Progress"] }, deleted_at: null })` |
| Total Departments | `departments` (non-deleted) | `countDocuments` |
| Total Employees | `employees` (non-deleted) | `countDocuments` |
| Assets Assigned | `assets` | `countDocuments({ status: "Assigned", deleted_at: null })` |
| Monthly Ticket Trend | `tickets` (last 6 months) | `aggregate` group by year+month |
| Tickets by Category | `tickets` → `ticket_categories` | `aggregate` with `$lookup` |

**Return type:** `DashboardData` interface with all metrics. The service function `getDashboardData()` accepts an optional `userRole` parameter for filtering.

## 2. Create Dashboard Server Action (`lib/actions/dashboard-actions.ts`)

```ts
"use server"
export async function getDashboardData(): Promise<DashboardData>
```

Calls `getDashboardData()` from the service. No auth check here (the page-level `PageGuard` handles access). Returns the aggregated data.

## 3. Rewrite Dashboard Page (`app/(dashboard)/dashboard/page.tsx`)

**Remove:** All mock data imports, hardcoded stats array.

**Add:** 
- `useEffect` to fetch data via `getDashboardData()` server action
- `useAuthorization()` hook for permission checks
- Loading skeleton state
- Conditional section rendering based on permissions

**Layout structure (role-aware):**

```
Header: "Dashboard" + subtitle
─────────────────────────────────
Row 1: Stat Cards (4-col grid)
  - Total Assets          → always show
  - Assigned Assets       → always show
  - Open Tickets          → always show (if user can view /tickets)
  - Total Departments     → always show
─────────────────────────────────
Row 2: Charts (2-col grid)
  Left:  Asset Status Distribution (PieChart / Donut)
  Right: Ticket Status Distribution (PieChart / Donut)
─────────────────────────────────
Row 3: Charts (2-col grid)
  Left:  Assets by Category (horizontal BarChart)
  Right: Monthly Ticket Trend (line or bar Chart)
─────────────────────────────────
Row 4: Recent Activity (3-col grid)
  Left (col-span-2): Recent Assets table
  Right: Quick Actions (permission-gated links)
```

**Permission gates:**
- Ticket sections (stat, chart, trend) → `hasPermission("/tickets", "View")`
- Asset sections → `hasPermission("/assets", "View")`
- Quick Actions: each link checks its target page's "View" permission
  - "View All Assets" → `/assets` View
  - "Add New Asset" → `/assets` Add
  - "View Reports" → `/ticket-report` View
  - "Manage Users" → `/users` View
