## Key Implementation Details

### Service Layer (`employee-service.ts`)
- `getEmployeeSelectOptions()` â€” returns departments list for form dropdown (pattern from `user-service.ts:54-66`)
- All queries use `.populate("department_id", "name")` to resolve FK
- `toEmployee()` mapper handles both populated (`{_id, name}`) and unpopulated (string) `department_id` (pattern from `user-service.ts:10-28`)

### Form Modal (`employee-form-modal.tsx`)
- Department dropdown uses `<Select>` component with "No Department" option
- Fetches departments via `getEmployeeSelectOptions()` on dialog open
- Required fields: `firstname`, `lastname`; all others optional
- Grid layout: Emp No + First Name, Middle Name + Last Name, Email + Contact No, Department

### Column Definitions
- Name column: combined `firstname` + `lastname` display
- Department column: shows `department_name` or "N/A"
- Status column: same `statusConfig` badge pattern (Active=emerald, Deleted=rose)
- Actions: View Details, Edit Employee (permission-gated), Delete/Restore (permission-gated)
