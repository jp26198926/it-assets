# Add Order Number Field to Pages

## Goal
Add an `order` number field to the Page entity, used for sorting pages in the sidebar (currently sorted by `created_at`).

## Files to Modify

### 1. Type Definition
**`lib/types/page.ts`**
- Add `order: number` to `Page` interface
- Add `order?: number` to `CreatePageInput`

### 2. Mongoose Schema
**`lib/db/models/page.ts`**
- Add `order: number` to `IPage` interface
- Add `order` field to schema: `{ type: Number, default: 0 }`
- Add index for sorting: `{ order: 1 }`

### 3. Service Layer
**`lib/services/page-service.ts`**
- Update `toPage()` to map `p.order`
- Update `createPage()` to pass `order`
- Update `updatePage()` to handle `order` field
- Update `getSidebarPages()`: change sort from `created_at: 1` to `order: 1`

### 4. API Routes
**`app/api/pages/route.ts`**
- POST: accept and pass `body.order`

**`app/api/pages/[id]/route.ts`**
- PUT: accept and pass `body.order`

### 5. Server Actions
**`lib/actions/page-actions.ts`**
- No changes needed (already passes through to service)

### 6. UI - Form Modal
**`components/modals/page-form-modal.tsx`**
- Add `order` to `defaultFormData` (default: `0`)
- Add order number input field in the form grid
- Include `order` in form submission

### 7. UI - View Modal
**`components/modals/page-view-modal.tsx`**
- Display order number in the detail grid

### 8. UI - Table Columns
**`components/data-table/page-data-table-columns.tsx`**
- Add `order` column (numeric, sortable) between Section and Parent columns

## Verification
1. Start dev server: `npm run dev`
2. Navigate to /pages, create a page with order number
3. Edit a page and verify order persists
4. Check the sidebar — pages should sort by order field
5. Verify table shows the order column
6. Run `npm run lint` and `npm run typecheck` to ensure no errors
