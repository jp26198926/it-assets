# Plan: Make Project API-Ready (Pages & Permissions)

## Goal
Create REST API routes as the primary data layer for Pages and Permissions. Extract shared DB logic into a service layer. Keep server actions as thin wrappers (zero component changes needed).

## Architecture

**Pattern:** Service Layer + Route Handlers + Thin Server Actions

```
Route Handler (app/api/*)  ──┐
                              ├──> Service Layer (lib/services/) ──> Mongoose Models
Server Action (lib/actions/) ─┘
```

**Response format:**
```json
{ "success": true, "data": <T> }
{ "success": false, "error": "message" }
```

## API Endpoints

### Pages
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/pages` | List with filters (`?search=&name=&path=&status=&section=&parent_id=`) |
| POST | `/api/pages` | Create (`{ name, path, icon, description?, parent_id?, section? }`) |
| GET | `/api/pages/[id]` | Get by ID |
| PUT | `/api/pages/[id]` | Update |
| DELETE | `/api/pages/[id]` | Soft-delete |
| POST | `/api/pages/[id]/restore` | Restore |
| GET | `/api/pages/statuses` | Get page statuses |
| GET | `/api/pages/sidebar` | Get active sidebar pages |

### Permissions
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/permissions` | List with filters (`?search=&name=&description=&status=`) |
| POST | `/api/permissions` | Create (`{ name, description? }`) |
| GET | `/api/permissions/[id]` | Get by ID |
| PUT | `/api/permissions/[id]` | Update |
| DELETE | `/api/permissions/[id]` | Soft-delete |
| POST | `/api/permissions/[id]/restore` | Restore |
| GET | `/api/permissions/statuses` | Get permission statuses |

## New Files to Create

```
lib/services/
├── api-helpers.ts           # apiSuccess(), apiError() response helpers
├── page-service.ts          # All page CRUD logic (extracted from page-actions.ts)
└── permission-service.ts    # All permission CRUD logic (extracted from permission-actions.ts)

app/api/pages/
├── route.ts                 # GET (list) + POST (create)
├── statuses/route.ts        # GET (statuses)
├── sidebar/route.ts         # GET (sidebar pages)
└── [id]/
    ├── route.ts             # GET + PUT + DELETE
    └── restore/route.ts     # POST (restore)

app/api/permissions/
├── route.ts                 # GET (list) + POST (create)
├── statuses/route.ts        # GET (statuses)
└── [id]/
    ├── route.ts             # GET + PUT + DELETE
    └── restore/route.ts     # POST (restore)
```

## Files to Modify

| File | Change |
|------|--------|
| `lib/actions/page-actions.ts` | Rewrite as thin wrappers calling page-service |
| `lib/actions/permission-actions.ts` | Rewrite as thin wrappers calling permission-service |

**No component changes needed** — page.tsx files and sidebar keep calling server actions as before.

## Implementation Order

1. Create `lib/services/api-helpers.ts` — response helpers
2. Create `lib/services/page-service.ts` — extract from page-actions.ts, deduplicate ObjectId mapping into a single `toPage()` helper
3. Create `lib/services/permission-service.ts` — extract from permission-actions.ts, same deduplication
4. Rewrite `lib/actions/page-actions.ts` as thin wrappers (1-line delegates)
5. Rewrite `lib/actions/permission-actions.ts` as thin wrappers
6. Create `app/api/pages/` route handlers (all 4 files)
7. Create `app/api/permissions/` route handlers (all 4 files)

## Validation

1. Run `npm run build` — ensure no type errors
2. Run `npm run dev` — test all API endpoints:
   - `GET /api/pages` returns page list
   - `POST /api/pages` creates a page
   - `GET /api/pages/[id]` returns single page
   - `PUT /api/pages/[id]` updates a page
   - `DELETE /api/pages/[id]` soft-deletes
   - `POST /api/pages/[id]/restore` restores
   - Same pattern for `/api/permissions/*`
3. Verify Pages page (`/pages`) still works (server actions → service)
4. Verify Permissions page (`/permissions`) still works
5. Verify Sidebar still loads navigation items
